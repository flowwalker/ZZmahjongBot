# 数据增强演进

> 4 个版本，288-fold增强：从通用API → 完整变换 → 随机采样 → 分层采样
>
> 但最后还是回到通用API乖乖一级一级增加做变换

---

## 小小创新思考

数据增强基于四个独立、可交换的变换因子：

```
Z₂ × S₃ × S₃ × Z₄ = 2 × 6 × 6 × 4 = 288 种变换

1. 数字对称 Z₂:     1↔9, 2↔8, 3↔7, 4↔6 (数字花色内), 5不变
2. 数字花色排列 S₃:  万/筒/条三色数字牌的全排列 (6种)
3. 箭牌排列 S₃:      中/发/白的全排列 (6种)
4. 风位旋转 Z₄:      东→南→西→北循环旋转 (4种)
```

四个因子可交换 → 任意组合都合法 → 288 种变换。适用条件：不涉及具体数字/花色的 action 均保持不变（这里就直接不考虑绿一色之类的罕见牌型了）

## v2: 288-fold 基础版

```
通用 288-fold 实现 → augment_batch(obs, mask, act)
```

从 155ch 特化解耦为通用实现，支持任意通道布局。提供 `augment_batch` 接口，接收 (obs_stack, mask_stack, act_stack)，返回增强后的 batch。

目的是将通道映射逻辑参数化。

---

## v3: 288-fold （最后都是以这个为基准）

相比 v2 的改进：

```
完整变换API: batch变换 + 单样本变换 + 逆变换 + 按 level 获取变换子集
```

**说明**:

- 单样本变换（`apply_transform`）与逆变换在 v2 已存在，v3 保留并整理接口。
- v3 新增 `get_transforms_by_level(level)`，可按 2×/12×/72×/288× 等层级获取变换子集。
- 不存在独立的「变换矩阵导出」验证函数；变换信息以内置 dict 形式传递。

---

## v4: 随机采样（效果不佳）

相比 v3 的改进：

```
get_random_transforms(n) → 从 288 池随机选 n 个变换
支持 seed 控制可重复性
```

**核心变化**: 不遍历全部 288 种变换，而是每 epoch 随机采样 n 个（典型 n=2~4）。每个样本用随机选中的变换增强。

全部 288 种变换训练计算量过大（288× 数据膨胀），随机采样似乎能够增加多样性，但是实际更菜了...应该学习更加不够稳定。

---

## v5: 分层随机采样（似乎不好）

相比 v4 的改进：

```
get_stratified_random_transform() → 每个因子组各选一个
数字对称(2选1) × 数字花色排列(6选1) × 箭牌排列(6选1) × 风位旋转(4选1)
```

从完全随机改为分层随机——每个变换因子独立随机选择，保证四个维度的多样性都被覆盖。
