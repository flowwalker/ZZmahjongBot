# Record

基于 v6 (148ch+BiLSTM+Attention) 的第二次迭代。核心的改动：特征工程 148→224ch + 架构微调。

---

## feature.py：148ch → 224ch（+76 通道）

```
224 = 148(原版) + 44(V2 语义) + 32(V3 专家)
```

设计理念：将人类麻将专家的牌效分析逻辑编码为神经网络输入特征。

**后来证明是比较失败的**，原因：

- 预处理、预计算时间过长
- 数据量翻倍，训练时间IO大幅翻倍，而此刻我还没有使用后来的“临时处理技巧”

  **在时间和金钱不足的双重打击下**放弃

  虽然失败，但还是认为值得一提创新的点子：

f2手工编码的麻将语义信息：

| 通道 | 名称 | 说明 |
|------|------|------|
| 148–151 | HAND_COUNT | 手牌张数阈值 (≥1/2/3/4) |
| 152–155 | EFFICIENCY | 牌效率评估 (搭子×进张×墙密度)，连续值+3阈值 |
| 156–159 | SAFETY | 安全度 (字牌筋×壁×剩余)，连续值+3阈值 |
| 160–163 | OPP_RISK | 对手危险度 (弃牌花色偏好+副露数) |
| 164–171 | YAKU_POTENTIAL | 8种番型各自打分(平和/对对/混一色/字一色/混老头/断幺/门前/杠) |
| 172–175 | WALL_DENSITY | 牌墙密度 (剩余/4连续值 + ≤1/≤2/=0三阈值) |
| 176–179 | POSITION | 位置进度 (总体%/自墙%/≤5%/庄家) |
| 180–187 | TEMPORAL | 时序弃牌统计 (每家最近7步/14步分布) |
| 188–191 | META | 听牌概率/手牌填充率/副露数/杠存在 |

f3进阶专家推理：

| 通道 | 名称 | 说明 |
|------|------|------|
| 192–195 | TILE_DRAW | 进张贡献 (模拟打出→摸牌后有效牌数) |
| 196–199 | SHANTEN | 向听数 (当前+打出后+≤1听阈值) |
| 200–203 | WAIT_TYPE | 听牌类型 (单骑/两面/对碰/边张) |
| 204–207 | DANGER | 精确危险度 (筋牌+现物+可见张综合) |
| 208–211 | YAKU_PROG | 番型进度 (平和/断幺/混一色/清一色) |
| 212–215 | HAND_STRUCT | 手牌结构 (对子/搭子/刻子/顺子 归一化) |
| 216–219 | OPP_INFER | 对手推理 (需要花色/倾向弃牌/筋安全/现物) |
| 220–223 | PHASE | 游戏阶段 (序盘/中盘/终盘/局末) |

## model.py 进行了微调

| 项目 | v6 | v7 |
|------|----|----|
| SE reduction | **16** | **8** (减少过度压缩) |
| Policy头 | FC→Mish→FC | FC→**LayerNorm**→Mish→**Dropout(0.05)**→FC |
| BiLSTM | 默认初始化 | forget gate bias 初始化=**1.0** |
| 参数量 | ~39M | ~158M |
