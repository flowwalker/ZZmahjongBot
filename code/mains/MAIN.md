# Botzone 部署演进

> 3 个版本，单模型 → 3模型投票 → 投票+TTA

---

## v1: 单模型推理

```
FeatureAgent(160ch) → CNN-SE Pyramid模型 → 单次推理 → argmax 出牌
```

最基础的部署模式。加载一个训练好的模型权重，每次需要决策时做一次前向传播，选取概率最大的合法动作。

单模型部署覆盖了整个第一、二阶段的实验需求——模型架构从 6ch CNN 一路演进到 160ch CNN-SE Pyramid，但部署代码的核心逻辑始终不变：加载模型→特征编码→推理→argmax。区别仅在于换模型权重文件和模型类名，因此所有单模型部署合并为一个代表性版本。

---

## v2: 3模型投票 + 100× TTA

```
3组同架构 CNNModel 模型（加载不同权重）× TTA(每组100次随机增强) → 全局软投票(平均概率) → argmax
```

**创新 — 多权重集成 + TTA**: 加载 3 组同架构但不同权重的 CNNModel，每次推理随机采样 100 个增强变换（从 288 变换池中），所有模型×变换的输出概率取全局平均后 argmax。通过权重集成与 TTA 增强鲁棒性。

相比 v1：从单模型单推理→多模型权重集成+TTA软投票，精度提升来源于权重集成与测试时增强。

---

## v3: 3模型投票 + 50× TTA (5s 预算)

```
3模型 × TTA(每模型50次随机增强) → 全局软投票 → argmax
5秒时间预算控制
```

**变化**: 将 TTA 次数从 100 降至 50，引入时间预算控制避免 Botzone 超时。核心逻辑与 v2 相同，区别仅在于 N_TTA 值和时间预算。

相比 v2：TTA 减半(50×3=150次推理)，牺牲少量精度换取时限安全。
