# 特征工程演进

## v1: 6ch

```
OBS_SIZE = 6
通道: 门风(1) + 圈风(1) + 手牌×4(堆叠编码)
```

仅包含门风、圈风和手牌（按持有张数堆叠）。无弃牌历史、无鸣牌信息、无牌墙信息。模型只能看到自己的手牌和当前场风。

---

## v2: 16ch — 加入场上可见信息

相比 v1 新增 10ch：

```
OBS_SIZE = 16
新增: 弃牌标记(4ch) + 鸣牌标记(4ch) + 明牌(1ch) + 牌墙剩余(1ch)
```

**动机**: v1 的 6ch 只有自手信息，模型无法感知场上局势。v2 引入了四家最近弃牌、鸣牌标记、手牌+明牌可见区域、牌墙剩余量，让模型具备了基本的场上感知能力。

---

## v3: 148ch — 弃牌历史

相比 v2，通道数从 16 跃升至 148：

```
OBS_SIZE = 148
通道布局:
  [0-3]   手牌(4ch) — 按张数分层编码
  [4-7]   明牌(4ch) — 手牌+场上明牌
  [8-23]  鸣牌(16ch) — 四家×4(吃/碰/明杠/暗杠)
  [24-27] 圈风(4ch) — one-hot
  [28-31] 门风(4ch) — one-hot
  [32-35] 牌墙(4ch) — 四家各自归一化
  [36-147] 弃牌序列(112ch) — 四家×28步，每步在34种牌矩阵打点
```

**核心创新 — 弃牌序列 4ch→112ch**: v2 只编码了每家最近一次弃牌（4ch），v3 将弃牌展开为完整历史轨迹：4家×28步=112ch。每一步在34种牌的矩阵上打点标记。模型从此能看到完整的弃牌历史，对判断对手听牌、计算牌安全度至关重要。

**动机**: 从"当前快照"到"历史轨迹"的质变。麻将中弃牌顺序蕴含了对手的手牌信息和战术意图，仅靠最新一次弃牌无法捕捉。

**其他改进**: 手牌从4ch直接堆叠改为1-4张分层编码；MELD从4ch细化为16ch（每家独立编码吃/碰/明杠/暗杠）。

---

## v4: 224ch — 语义专家特征（膨胀后被放弃）

相比 v3，通道数从 148 → 224（+76ch），是特征工程的膨胀顶点：

```
OBS_SIZE = 224 = 148(基础) + 44(语义特征) + 32(专家特征)
语义特征(44ch): 手牌效率(向听数)、牌安全度、役种潜力、对手建模
专家特征(32ch): 进张贡献、听牌型(单骑/两面/双碰)、危险度精确估计
```

**动机是**: 试图将人类牌理和专家知识直接编码为特征通道，让模型不需要从数据中学习这些先验知识。

**最终放弃**:

1. 计算代价太大了——每次更新观测都要实时计算番型
2. （从结果反推）特征冗余——CNN+SE 模型本身能从更少通道学到类似表示
3. 训练超级不稳定——224ch 输入增大模型参数量，反而更难收敛
4. 边际收益低——实际精度提升远不及计算成本增长

**结论**: 后续全部回退到更小的特征版本，确认了"直接可观测信息"路线优于"手工语义特征"路线。

---

## v5: 155ch — 辅助决策通道

相比 v3 新增 7ch

```
OBS_SIZE = 155 = 148 + 7
新增:
  [148]     当前待决策牌(1ch) — one-hot
  [149]     牌来源方(1ch) — 归一化0~1
  [150]     牌墙最后一张(1ch) — bool
  [151]     刚杠标志(1ch) — 杠上开花/抢杠胡判定
  [152-154] 对手手牌数(3ch) — 三家归一化/14
```

**动机**: 这 7ch 都是状态辅助信息而非麻将知识。显式编码能降低模型的推断负担。

---

## v6: 160ch 

相比 v5，OBS_SIZE 从 155 → 160（+5ch），核心变化是 MELD 编码从 16ch → 21ch：

```
OBS_SIZE = 160 = 155 + 5
MELD 变化: 16ch → 21ch
  旧(16ch): 每家4ch (吃/碰/明杠/暗杠) × 4家
  新(21ch): 每家5ch (吃左/吃中/吃右/碰/明杠) × 4家 + 自暗杠1ch
其余通道因 MELD 偏移+5 整体后移，内容不变
```

**动机 — 吃牌方向编码**: 吃牌细分为左吃(如"23吃1")/中吃(如"13吃2")/右吃(如"12吃3")，

此版本是最终统一采用的特征，后续所有部署均基于 160ch。160ch 是最终采用的特征维度。
