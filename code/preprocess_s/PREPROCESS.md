# 数据预处理演进

> 4 个版本，单线程 → 多进程 → 增强内嵌 → 分离为王

---

## v1: 单线程标准版

```
逐文件读取 data.txt → 逐局解析 → 逐样本 feature化 → np.savez
单进程，顺序执行
```

最基础的预处理实现。读取原始 `data.txt`，按 Match 行切分对局，逐局调用 FeatureAgent 将游戏状态转为特征向量，过滤无效样本（action_mask ≤1），保存为 `.npz` 文件。

**性能**: 单线程，I/O 和特征计算串行，处理速度受限于单核 CPU。

---

## v2: 多进程版

相比 v1 的改进：

```
虚拟切分(seek) → 多进程Pool → 每进程独立处理一个Part → 合并count.json
NUM_WORKERS = cpu_count() - 2
```

目的当然是加速。

**创新**:

1. **虚拟切分**: 不物理切分 `data.txt`，而是扫描所有 Match 的字节偏移量，多进程通过 seek 直接跳到各自区间
2. **内存缓存**: 启动时预加载已存在文件列表到 set，避免循环内反复 `os.path.exists`
3. **并行处理**: 使用 `multiprocessing.Pool`，N-2 个 worker 并行处理

**动机**: 单线程处理大规模数据耗时过长（数百万对局），多进程虚拟切分免去物理切分文件的 I/O 开销。

---

## v3: 增强内嵌版（失败）

相比 v1 的改进：

```
单线程 → 特征化 → 嵌入数据增强 → np.savez
```

**变化**: 在预处理阶段直接嵌入数据增强逻辑，将增强后的样本一并保存。

**问题**: 增强后磁盘占用爆炸，且增强策略与预处理耦合，无法灵活调整。

---

## v4: Raw-Only 多进程版

上面那些在数十倍增强面前根本无用，存多，IO多，时间慢。

因此解耦合，回归。

```
多进程 → 纯特征化 → savez (无增强)
增强移至 Dataset 层 (延迟增强/LazyAug)
```

**为了解决数据增强后磁盘爆炸的问题，唯一解是回到 — 增强解耦**: 预处理回归纯粹——只做原始数据的特征化转换，不再嵌入增强。增强移至 Dataset 层以 LazyAug 形式动态执行。同时在 v2 多进程基础上保持并行处理能力。

**动机**:

1. **解耦**: 预处理和增强职责分离，各自可独立迭代

2. **灵活性**: 增强策略可在训练时动态调整，无需重跑预处理

3. **存储**: 预处理产出是原始特征，**现在解耦的真实目的是**使得增强在内存中即时变换，不占用额外磁盘空间⚠️⚠️这是最最重要的一点

   但是一个巨大的要求的是大内存空间，比如Mac本地32G内存就支持不了了，利用华为算力才可。
